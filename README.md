# 🎓 ระบบติดตามแผนงานและโครงการ (Executive Dashboard) - โรงเรียนครุศาสตร์

แดชบอร์ดสรุปและวิเคราะห์ผลการติดตามแผนงานและโครงการสำหรับผู้บริหารโรงเรียน/คณะครุศาสตร์ โดยดึงข้อมูลแบบเรียลไทม์ (Real-time Live Sync) จาก Google Sheets ของหน่วยงานโดยตรง ทำให้ข้อมูลอัปเดตใหม่เสมอในทุกครั้งที่มีผู้เข้าชมหรือกดปุ่มรีเฟรชหน้าเว็บ

---

## 🌟 ฟีเจอร์หลัก (Key Features)

1. **การซิงก์ข้อมูลแบบสด (Live Google Sheets Integration)**:
   - ดึงข้อมูลอัตโนมัติจาก Google Sheets ผ่าน Fetch API โดยไม่มีการเก็บแคชฝั่งเซิร์ฟเวอร์
   - ข้อมูลจะถูกดึงใหม่ทุกครั้งที่เข้าหน้าเว็บ หรือเมื่อกดปุ่ม "รีเฟรชข้อมูล" (มีระบบหมุนโหลดแบบอนิเมชันบ่งบอกการซิงก์)
2. **สรุปตัวชี้วัดสำคัญ (Executive KPI Cards)**:
   - สรุปจำนวนโครงการทั้งหมด (รวมถึงจำนวนที่ดำเนินการอยู่และสำเร็จแล้ว)
   - สรุปงบประมาณจัดสรร, งบประมาณที่ใช้ไปแล้ว, และงบประมาณคงเหลือพร้อมเปอร์เซ็นต์อัตราการใช้จ่าย
   - คำนวณ **"ความคืบหน้าเฉลี่ยถ่วงน้ำหนัก (Weighted Average Progress)"** ตามสัดส่วนงบประมาณของโครงการจริง (สะท้อนผลการดำเนินงานที่ถูกต้องทางวิชาการและระบบบริหารความเสี่ยง)
3. **การนำเสนอข้อมูลด้วยกราฟ (Data Visualization)**:
   - **งบประมาณเปรียบเทียบกับใช้จ่ายจริง**: กราฟแท่งคู่เปรียบเทียบในแต่ละโครงการ
   - **สัดส่วนงบประมาณจัดสรรตามกลุ่มงาน**: กราฟโดนัทแสดงสัดส่วนงบประมาณของวิชาการ, วิทยาศาสตร์และเทคโนโลยี, สังคมศึกษา, งบประมาณและสินทรัพย์ ฯลฯ
   - **ความคืบหน้าของโครงการรายบุคคล**: กราฟแท่งแนวนอนแสดงความสำเร็จ (%) ของแต่ละโครงการ
   - **จำนวนโครงการจัดสรรตามผู้รับผิดชอบ**: กราฟวงกลมจำแนกสัดส่วนงานโครงการของผู้รับผิดชอบแต่ละท่าน
4. **ระบบค้นหาและกรองข้อมูลขั้นสูง (Interactive Filtering & Sorting)**:
   - ค้นหาโครงการแบบเรียลไทม์จากชื่อโครงการหรือชื่อผู้รับผิดชอบ (Live Search)
   - ตัวกรองแบบ Dropdown แยกตาม **กลุ่มงาน**, **ผู้รับผิดชอบหลัก**, และ **สถานะโครงการ**
   - การคลิกที่หัวตารางเพื่อเรียงลำดับข้อมูล (Sorting) ทั้งแบบก้าวหน้าและถอยหลัง (เช่น เรียงตามงบประมาณสูง-ต่ำ)
5. **การแสดงรายละเอียดเชิงลึก (Project Detail Modal)**:
   - แสดงแผนทางการเงิน สถิติเบิกจ่าย และแถบเปอร์เซ็นต์งบประมาณของโครงการนั้นๆ
   - แสดงเกจความคืบหน้าเป็นวงกลมแบบมีดีไซน์สวยงาม (SVG Circular Progress Gauge)
6. **ธีมและการออกแบบระดับพรีเมียม (Premium Aesthetics & Theme Switcher)**:
   - ธีมสีครุศาสตร์ (สีส้มแสด/ส้มอิฐ) ผสานงานออกแบบ Glassmorphism ไร้ขอบ มีมิติ
   - สลับโหมดการแสดงผล (Light / Dark Mode) ได้ผ่านปุ่มเดียว โดยระบบจะทำการบันทึกค่าไว้ใน `localStorage`
   - ระบบชาร์ต (Chart.js) จะปรับเปลี่ยนโทนสี เส้นตาราง (Gridlines) และสีอักษรให้อ่านง่ายสอดคล้องกับธีมที่เลือกโดยอัตโนมัติ

## 📂 โครงสร้างโปรเจกต์ (Project Files)

- `index.html` - โครงสร้างหลักและคอมโพเนนต์ต่าง ๆ ของ Dashboard
- `style.css` - สไตล์ดีไซน์ระบบ Vanilla CSS3 ทั้งหมด (พร้อม Grid, Flexbox, Custom scrollbars และ Responsive styles)
- `app.js` - ลอจิกควบคุมการ Fetch, ล้างตัวเลข, คัดกรอง, จัดกลุ่มเพื่อสร้างชาร์ต, และงานโต้ตอบ UI ทั้งหมด
- `README.md` - คำอธิบายรายละเอียดโปรเจกต์และวิธีอัปโหลดขึ้น GitHub Pages

## 🚀 วิธีอัปโหลดขึ้น GitHub และเปิดใช้งาน GitHub Pages

เพื่อเผยแพร่ Dashboard นี้ให้ผู้บริหารและผู้สนใจเข้าใช้ผ่านลิงก์สาธารณะ คุณสามารถเปิดใช้งานบริการฟรีของ **GitHub Pages** ได้ตามขั้นตอนดังนี้:

### 1. สมัครใช้งานและติดตั้ง Git (หากยังไม่มี)
- สมัครสมาชิกที่ [GitHub](https://github.com/)
- ดาวน์โหลดและติดตั้ง [Git](https://git-scm.com/) ลงบนเครื่อง

### 2. เตรียม Git Repository บนโฟลเดอร์ของระบบ
เปิดโปรแกรม Terminal หรือ PowerShell ในโฟลเดอร์นี้ แล้วรันคำสั่ง:
```bash
# 1. เริ่มต้นระบบกิตในโฟลเดอร์โครงการ
git init

# 2. เพิ่มไฟล์ทั้งหมดเข้าสู่สเตจเตรียมบันทึก
git add .

# 3. บันทึกประวัติครั้งแรก (Initial Commit)
git commit -m "Initial commit of school dashboard"
```

### 3. สร้าง Repository บน GitHub และเชื่อมต่อ
1. ลงชื่อเข้าใช้ GitHub แล้วคลิกปุ่ม **New** (หรือเครื่องหมาย `+` มุมขวาบน) เพื่อสร้าง Repository ใหม่
2. ตั้งชื่อ Repository เช่น `school-project-dashboard`
3. ตั้งค่าเป็น **Public** (จำเป็นสำหรับใช้งาน GitHub Pages ฟรี) และห้ามทำเครื่องหมายเลือก Add README.md หรือ .gitignore (เพราะเรามีอยู่แล้ว) จากนั้นกด **Create repository**
4. ทำการคัดลอกคำสั่งสำหรับเชื่อมโยงในหน้าเว็บ GitHub (ส่วน "or push an existing repository from the command line") เช่น:
```bash
# เชื่อมโยงโฟลเดอร์ภายในเครื่องกับ Repository บนคลาวด์
git remote add origin https://github.com/USERNAME/school-project-dashboard.git

# เปลี่ยนชื่อกิ่งหลักเป็น main
git branch -M main

# อัปโหลดโค้ดขึ้น GitHub
git push -u origin main
```
*(แทนที่ `USERNAME` ด้วยชื่อบัญชี GitHub และ `school-project-dashboard` ด้วยชื่อโครงการที่คุณสร้าง)*

### 4. เปิดใช้งาน GitHub Pages เพื่อสร้างลิงก์เว็บไซต์สาธารณะ
1. ไปที่หน้า Repository ของคุณบนเว็บไซต์ GitHub
2. คลิกแถบ **Settings** (รูปฟันเฟือง) ด้านบนสุด
3. ที่เมนูด้านซ้าย เลือกหัวข้อ **Pages** (ภายใต้หมวดหมู่ Code and automation)
4. ในส่วน **Build and deployment** -> **Source** ให้เลือกเป็น **Deploy from a branch**
5. ในช่อง **Branch** ให้เลือกกิ่งเป็น **main** และเลือกโฟลเดอร์เป็น `/ (root)` จากนั้นคลิกปุ่ม **Save**
6. รอประมาณ 1-2 นาที จากนั้นให้รีเฟรชหน้าการตั้งค่า Pages จะมีข้อความแสดงสีเขียวพร้อมลิงก์เว็บไซต์ของคุณด้านบนสุด เช่น:
   `Your site is live at https://USERNAME.github.io/school-project-dashboard/`
